## Prerequisites

Before publishing, make sure your plugin:

- тЬЕ Exports valid `meta` (with `id`, `name`, `version`) and `setup`
- тЬЕ Exports `teardown()` for cleanup (removing CSS, clearing intervals, etc.)
- тЬЕ Uses plugin-scoped storage (`getForPlugin`/`setForPlugin`)
- тЬЕ Has no external dependencies (vanilla JS only)
- тЬЕ Works in modern browsers
- тЬЕ Handles the `plugin:unload` event or exports `teardown()`

## Step 1: Host Your Plugin

Upload `plugin.js` somewhere publicly accessible. GitHub raw URLs are recommended:

```
https://raw.githubusercontent.com/youruser/your-repo/main/plugin.js
```

<Note>
  The URL must be directly accessible тАФ not behind a login or download page. The core fetches the file directly via `fetch()`.
</Note>

## Step 2: Fork the Community Repo

Fork [Empty_Plugins](https://github.com/dheeraz101/Empty_Plugins) on GitHub.

## Step 3: Add Your Entry

Edit `plugins.json` and add:

```json
{
  "id": "your-plugin-id",
  "name": "Your Plugin Name",
  "url": "https://raw.githubusercontent.com/youruser/your-repo/main/plugin.js",
  "description": "A short description of what your plugin does.",
  "author": "your-github-username",
  "icon": "ЁЯзй"
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `id` | тЬЕ | Unique kebab-case identifier |
| `name` | тЬЕ | Display name |
| `url` | тЬЕ | Public URL to the `.js` file (must end with `.js`) |
| `description` | тЬЕ | What the plugin does |
| `author` | тЬЕ | Your GitHub username |
| `icon` | тЭМ | Emoji icon (defaults to ЁЯУж) |

## Step 4: Open a Pull Request

1. Commit your changes
2. Push to your fork
3. Open a PR against `main`
4. Use the title: `add: your-plugin-name`

## Plugin Checklist

- тЬЕ **Self-contained** тАФ use the event bus, not imports
- тЬЕ **Clean up** тАФ export `teardown()` that removes CSS, clears intervals, removes event listeners
- тЬЕ **Scoped storage** тАФ use `getForPlugin`/`setForPlugin`
- тЬЕ **Mobile-friendly** тАФ test on smaller screens
- тЬЕ **Valid meta** тАФ includes `id`, `name`, `version`
- тЬЕ **No eval()** тАФ security risk
- тЬЕ **No undisclosed API calls** тАФ respect privacy
- тЬЕ **CORS-compatible** тАФ your hosting must allow cross-origin fetches

## Example Template

```javascript
export const meta = {
  id: 'your-plugin-id',
  name: 'Your Plugin Name',
  version: '1.0.0',
  compat: '>=3.3.0'
};

let currentApi = null;

export function setup(api) {
  currentApi = api;

  api.injectCSS(meta.id, `
    .ypw-container {
      background: #1a1a2e;
      color: #eee;
      padding: 20px;
      border-radius: 12px;
    }
  `);

  const container = api.container;
  container.innerHTML = `
    <div class="ypw-container">
      <h3>Your Plugin</h3>
      <p>Description here.</p>
    </div>
  `;

  // Restore position
  const pos = api.storage.getForPlugin(meta.id, 'pos');
  if (pos) {
    container.style.left = pos.left + 'px';
    container.style.top = pos.top + 'px';
  }

  // Save position on drag
  api.bus.on('plugin:dragend', ({ el }) => {
    if (el.dataset.pluginId === meta.id) {
      api.storage.setForPlugin(meta.id, 'pos', {
        left: parseInt(el.style.left),
        top: parseInt(el.style.top)
      });
    }
  });
}

export function teardown() {
  currentApi?.removeCSS(meta.id);
}
```

## After Approval

Your plugin will appear in the **Community Store** tab of the Plugin Manager for all Blank Board users.
